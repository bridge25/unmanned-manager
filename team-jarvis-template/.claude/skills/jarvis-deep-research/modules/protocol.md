# Protocol Module - 4-Phase Sequential Research

## Overview

Chain of Density + Tree of Thoughts 기법을 결합한 순차적 리서치 프로토콜.
각 Phase의 출력이 다음 Phase의 컨텍스트가 되어 깊이를 더함.

---

## Phase 1: 구조적 해체 (Deconstruction)

**목적**: 주제를 분해하여 명확한 분석 기반 마련

### 수행 항목

#### 1.1 핵심 키워드 정의
```markdown
- **주요 개념**: [주제의 핵심이 되는 개념 정의]
- **관련 용어**: [혼용되거나 관련된 용어들]
- **분석 범위**: [이번 리서치에서 다룰 범위 명시]
- **제외 범위**: [다루지 않을 영역 명시]
```

#### 1.2 현재 상태 (Status Quo)
```markdown
- **시장 규모**: [현재 시장 크기, 성장률]
- **주요 플레이어**: [상위 3-5개 기업/서비스]
- **점유율**: [시장 점유율 분포]
- **기술 성숙도**: [초기/성장/성숙/쇠퇴]
- **규제 현황**: [관련 법규, 정책]
```

#### 1.3 시의성 (Timeliness)
```markdown
- **왜 지금인가**: [현재 이 주제가 중요한 이유]
- **트리거 이벤트**: [최근 발생한 중요 변화]
- **윈도우 오브 기회**: [기회의 창이 열려있는 기간]
- **늦으면 어떻게 되나**: [타이밍 놓칠 시 리스크]
```

### Phase 1 품질 체크리스트
- [ ] 모호한 용어 없이 명확하게 정의했는가?
- [ ] 데이터로 현황을 뒷받침했는가?
- [ ] "왜 지금"에 대한 답이 명확한가?

## Phase 2: 다각적 분석 (Multi-Lens Analysis)

**목적**: 4가지 렌즈로 입체적 분석, 편향 방지

### 2.1 Macro Lens (거시적 관점)

```markdown
#### 경제적 영향
- 시장 규모 변화
- 가격 구조
- 경제적 파급 효과

#### 규제 환경
- 현행 규제
- 예상되는 규제 변화
- 규제 리스크

#### 사회적 파급력
- 사회적 수용도
- 문화적 영향
- 윤리적 고려사항
```

### 2.2 Micro Lens (미시적 관점)

```markdown
#### 기술적 세부사항
- 핵심 기술 요소
- 기술 스택
- 구현 복잡도

#### 사용자 경험
- 사용자 여정
- 페인 포인트
- 가치 제안

#### 실무적 병목
- 운영상 어려움
- 스케일링 이슈
- 유지보수 부담
```

### 2.3 Bull Case (기회/긍정)

```markdown
#### 최상 시나리오
- 이상적 결과
- 달성 조건

#### 성공 트리거
- 성공을 위한 핵심 요인
- 촉매 이벤트

#### 상승 여력
- 성장 잠재력
- 확장 가능성
```

### 2.4 Bear Case (위험/부정)

```markdown
#### 실패 요인
- 실패 가능성 높은 시나리오
- 피해야 할 실수

#### 숨겨진 리스크
- 표면에 드러나지 않는 위험
- 간과하기 쉬운 요소

#### 비판적 시각 (Devil's Advocate)
- 반대 의견
- 회의적 시각의 근거
```

### Phase 2 품질 체크리스트
- [ ] 4가지 렌즈 모두 균형있게 분석했는가?
- [ ] Bull/Bear가 편향되지 않았는가?
- [ ] 각 관점에 근거가 있는가?

## Phase 3: 심층 통합 (Deep Synthesis)

**목적**: Phase 2 정보들의 연결고리 발견, 2차 효과 분석

### 3.1 Trade-offs 분석

```markdown
#### 충돌 지점
| 관점 A | vs | 관점 B | 해결 방향 |
|--------|----|---------| ---------|
| [예: 빠른 성장] | ↔ | [품질 유지] | [선택 기준] |

#### 양립 불가능한 선택
- [선택지 A 선택 시 포기해야 하는 것]
- [선택지 B 선택 시 포기해야 하는 것]
```

### 3.2 2차 파급 효과 (Second-order Effects)

```markdown
#### 직접 효과 → 2차 효과 → 3차 효과
1. [직접 효과]: ...
   └→ [2차 효과]: ...
      └→ [3차 효과]: ...

#### 예상치 못한 결과
- [의도하지 않은 긍정적 결과]
- [의도하지 않은 부정적 결과]
```

### 3.3 숨겨진 상관관계

```markdown
#### 비직관적 연결
- [A와 B가 연결되어 있는 이유]
- [데이터 근거]

#### 패턴 발견
- [반복되는 패턴]
- [유사 사례와의 공통점]
```

### Phase 3 품질 체크리스트
- [ ] 단순 나열이 아닌 연결/통합을 했는가?
- [ ] 2차 효과까지 고려했는가?
- [ ] 논리적 추론에 비약이 없는가?

## Phase 4: 실행 판정 (Actionable Verdict)

**목적**: 사용자 맥락 기반 실행 가능성 판정 및 구체적 액션

### 4.1 사용자 맥락 필터

```markdown
| 항목 | 평가 | 비고 |
|------|------|------|
| **초기 비용** | $[금액] | [세부 내역] |
| **1인 실행** | Y/N | [이유] |
| **자동화 범위** | [X]% | [자동화 가능 영역] |
| **기술 적합성** | Y/N | [필요 기술 vs 보유 기술] |
| **첫 수익까지** | [기간] | [마일스톤] |
| **예상 ROI** | [X]% | [계산 근거] |
| **리스크 레벨** | L/M/H | [주요 리스크] |
```

### 4.2 최종 판정

```markdown
### 🎯 판정: [Go / No-Go / Conditional]

**Go**: 진행 권장
**No-Go**: 진행 비권장
**Conditional**: 조건부 진행 (조건 명시)

#### 판정 근거 (3줄)
1. [가장 중요한 근거]
2. [두 번째 근거]
3. [세 번째 근거]

#### Conditional일 경우 조건
- [충족해야 할 조건 1]
- [충족해야 할 조건 2]
```

### 4.3 구체적 액션

```markdown
### ⚡ 다음 액션 (내일 당장 할 수 있는 것)

1. **[액션 1]**
   - 소요 시간: [X분/시간]
   - 필요 리소스: [없음 / 있음: 상세]
   - 예상 결과: [...]

2. **[액션 2]**
   - 소요 시간: [X분/시간]
   - 필요 리소스: [...]
   - 예상 결과: [...]

3. **[액션 3]**
   - 소요 시간: [X분/시간]
   - 필요 리소스: [...]
   - 예상 결과: [...]
```

### Phase 4 품질 체크리스트
- [ ] 사용자 상황에 맞는 판정인가?
- [ ] 근거가 명확한가?
- [ ] 액션이 구체적이고 실행 가능한가?

## Protocol Execution Tips

### 1. 순차 실행의 중요성
- 각 Phase 완료 후 다음 Phase로
- 앞 Phase의 결과를 다음 Phase의 컨텍스트로 활용

### 2. 재귀적 심화
- 특정 영역이 중요하면 해당 부분만 다시 4-Phase 적용
- 예: Bull Case가 복잡하면 Bull Case만 별도 심층 분석

### 3. 토큰 효율화
- 복잡한 주제는 Phase 1-2와 Phase 3-4를 분리 실행
- `/clear` 후 이전 Phase 요약만 컨텍스트로 전달
