## 🚀 시작하기

### 첫 세션

```
와썹
```

이 한마디로 JARVIS가 활성화됩니다:
- 📊 프로젝트 현황 브리핑
- ✅ 오늘 할 일 정리
- ⚠️ 긴급 알림 표시
- 💡 추천 액션 제안

### 매일 아침 루틴

```
1. "와썹" → 전체 브리핑 받기
2. 우선순위 확인 → 오늘 집중할 것 결정
3. 작업 시작 → JARVIS와 함께 진행
4. 종료 전 → "오늘 뭐 했더라?" → 기록
```

## 🔬 Skills 가이드

### 🔍 Deep Research

**심층 리서치 - 재귀적 탐구 루프**

탐정처럼 단서를 따라가며, 과학자처럼 가설을 검증합니다.

#### 사용법

```bash
# 기본 리서치
/research AI 개발 외주 시장 트렌드

# 심층 모드 (수렴까지 계속)
/research AI 에이전트 사업성 --deep

# 빠른 스캔 (3회 반복 후 정리)
/research 경쟁사 현황 --quick
```

#### 동작 원리

```
PROBE (탐침)
    ↓ "지형이 어떻게 생겼지?"
DISCOVER (발견)
    ↓ "어? 이게 뭐지?"
DECIDE (결정)
    ↓ DIVE(깊이) / BRANCH(분기) / PRUNE(버림)
CONNECT (연결)
    ↓ "이 발견들이 어떻게 연결되지?"
CONVERGE (수렴)
    ↓ "본질에 도달했나?"
SYNTHESIZE (종합)
    → 실행 가능한 결론
```

#### 활용 시나리오

| 상황 | 사용법 | 옵션 |
|------|--------|------|
| 새 사업 아이템 검토 | `/research [아이템] 사업성` | `--deep` |
| 경쟁사 분석 | `/research [분야] 경쟁사 현황` | 기본 |
| 기술 스택 결정 | `/research [기술] vs [기술]` | 기본 |
| 시장 조사 | `/research [시장] 2024 트렌드` | `--deep` |
| 빠른 팩트체크 | `/research [주장] 근거` | `--quick` |

#### 결과물 예시

```markdown
# 🔍 Deep Research: AI 코딩 에이전트 시장

## 본질 (The Core)
**한 문장**: AI 코딩 시장은 도구(Copilot)에서 에이전트(Devin)로 진화 중이며,
             B2B 엔터프라이즈가 블루오션

## 핵심 발견
1. [데이터] GitHub Copilot 100만+ 유료 사용자
2. [트렌드] 에이전시 → 자동화 → AI 에이전트
3. [기회] 버티컬 특화 에이전트 부재

## 실행 판정
**Go**: B2B 버티컬 특화로 진입 권장
**다음 액션**:
1. 타겟 버티컬 3개 선정
2. PMF 인터뷰 10건
```

### 💡 Chaos Engine

**창의적 발상 - 통제된 카오스**

막혀있을 때, 새로운 관점이 필요할 때 사용합니다.

#### 사용법

```bash
# 기본 아이디어 발상
/chaos 새로운 SaaS 아이템

# 최대 창의성 모드
/chaos 차별화 전략 --wild

# 특정 제약 조건 포함
/chaos 3개월 안에 월 1000만원 만드는 방법
```

#### 동작 원리

```
CHAOS (발산)
    ↓ 10+ 아이디어 무작위 생성
HARVEST (수확)
    ↓ 유망한 것 선별
USER GATE (확인)
    ↓ "이 방향 맞아요?"
BRANCH (분기)
    ↓ Tree of Thought 평가
EVALUATE (평가)
    ↓ 실현 가능성 점수화
GROUND (착지)
    → 현실적 실행 계획
```

#### 활용 시나리오

| 상황 | 명령어 | 기대 결과 |
|------|--------|----------|
| 새 사업 아이디어 | `/chaos 부업 아이디어` | 10+ 아이디어 + 평가 |
| 마케팅 전략 | `/chaos 바이럴 마케팅` | 창의적 캠페인 안 |
| 문제 해결 | `/chaos [문제] 해결법` | 색다른 접근법 |
| 피벗 방향 | `/chaos 피벗 옵션` | 대안 시나리오 |
| 차별화 | `/chaos 경쟁자와 다른 방법` | USP 발굴 |

#### 결과물 예시

```markdown
# 💡 Chaos Engine: 3개월 수익화 방법

## 발산된 아이디어 (10개)
1. 🟢 AI 자동화 대행 (난이도: 낮음)
2. 🟢 GPT 래퍼 SaaS (난이도: 중)
3. 🟡 개발 컨설팅 (시간 교환)
4. 🟡 템플릿 판매 (패시브)
...

## Tree of Thought 평가
| 아이디어 | 실현성 | 수익성 | 속도 | 총점 |
|---------|:------:|:------:|:----:|:----:|
| AI 대행 | 9 | 7 | 9 | 25 |
| GPT SaaS | 7 | 8 | 6 | 21 |

## 착지 (Ground)
**추천**: AI 자동화 대행 + GPT SaaS 병행
**1주차 액션**:
- 대행: 포트폴리오 3개 준비
- SaaS: MVP 기획
```

콘텐츠 생성부터 SNS 게시까지 자동화합니다.

| 채널 | 톤앤매너 | 포맷 |

## 🎛️ /pm 오케스트레이션

**여러 프로젝트를 병렬로 관리**

PM(Project Manager)으로서 여러 Worker Claude에게 작업을 위임합니다.

#### 기본 사용법

```bash
# 기본 형식
/pm <프로젝트> <지시>

# 예시
/pm 프레젠토 테스트 실행해줘
/pm 해동검도 git status 확인
/pm 사주 README 첫 10줄 보여줘
```

#### 등록된 프로젝트 (예시)

| 세션명 | 프로젝트 | 별칭 |
|--------|---------|------|
| prezento | daddy's-ppt | 프레젠토 |
| haedong | order-automation-saas | 해동, 해동검도 |
| saju | saju | 사주 |
| qrparking | qr-parking-demo | 주차, parking |

#### 고급 기능

```bash
# 여러 프로젝트 동시 지시
/pm 프레젠토 테스트 실행
/pm 해동검도 빌드 확인
/pm 사주 린트 검사

# 결과 수집
/pm results  # 모든 결과 확인

# 세션 관리
/pm list     # 세션 목록
/pm status   # 전체 상태
```

#### LLM 라우팅 (Codex / Gemini)

`/pm` 지시에 특정 키워드가 포함되면 해당 LLM CLI가 자동 호출됩니다:

| 키워드 | 호출되는 CLI | 용도 |
|--------|-------------|------|
| `codex`, `/codex`, `코덱스` | `codex exec` | 코드 점검/디버깅/보안 |
| `gemini`, `/gemini`, `제미나이` | `gemini --yolo` | 2M 컨텍스트 분석 |

**예시:**
```bash
/pm 프레젠토 codex로 코드 리뷰해줘
/pm 해동검도 gemini로 전체 아키텍처 분석해줘
```

##### ⚠️ 선행 조건 (필수)

Codex/Gemini 기능을 사용하려면 **각자 CLI 설치 + API 키 설정**이 필요합니다:

**Codex (OpenAI):**
```bash
# 설치
npm install -g @openai/codex

# API 키 설정 (~/.bashrc 또는 ~/.zshrc)
export OPENAI_API_KEY="sk-..."
```

**Gemini (Google):**
```bash
# 설치
npm install -g @anthropic-ai/gemini-cli
# 또는 Google 공식 CLI 사용

# API 키 설정
export GOOGLE_API_KEY="..."
```

> 💡 **참고**: API 키 없이도 `/pm` 기본 기능(Worker Claude 위임)은 정상 작동합니다. Codex/Gemini는 선택적 고급 기능입니다.

#### 설정 방법

`config.yaml`에 프로젝트 등록:

```yaml
projects:
  prezento:
    path: "../daddy's-ppt"
    aliases: ["프레젠토", "ppt"]
  haedong:
    path: "../order-automation-saas"
    aliases: ["해동", "해동검도"]
```

## 🧠 Memento 시스템

**의사결정 기억 - Case Bank**

과거의 결정을 기억하고, 일관된 판단을 돕습니다.

#### 자동 기록되는 것

| 유형 | 예시 | 활용 |
|------|------|------|
| **의사결정** | "A vs B 선택" | 유사 상황에서 참조 |
| **성공 케이스** | "X 방법이 효과적" | 패턴 재사용 |
| **실패 케이스** | "Y 방법 실패" | 같은 실수 방지 |
| **가치 판단** | "Z가 더 중요" | 우선순위 일관성 |

#### 활용 방법

```bash
# JARVIS가 자동으로
# - 유사한 과거 케이스 검색
# - 과거 결정 인용하여 조언
# - 일관된 판단 기준 유지

# 수동 검색
"[주제] 관련 과거에 어떻게 했더라?"
"예전에 비슷한 결정 뭐 있었어?"
```

#### 저장 위치

```
current/cases.md  # 의사결정 케이스
current/profile.md  # 개인 패턴/선호
```

## 🔥 고급 활용법

### 1. 병렬 실행 체제

**순차가 아닌 병렬로 생각하기**

```
❌ 나쁜 예:
1. 도메인 구매 → 2. 설정 → 3. 배포 → 4. 마케팅

✅ 좋은 예:
[Track A: 인프라] 도메인 + 설정 (30분)
[Track B: 개발] 기능 완성 (메인) ← 동시 진행
[Track C: 마케팅] 콘텐츠 준비 (틈틈이)
```

**JARVIS에게 요청:**
```
"이 작업들 병렬로 정리해줘"
"동시에 할 수 있는 것들 묶어줘"
```

### 2. 공부 모드

**백그라운드 리서치 자동화**

```bash
# 트리거 키워드
"공부하고 있어"
"리서치 해둬"
"알아봐둬"

# 동작
1. research_queue.md에서 대기 주제 확인
2. 백그라운드에서 리서치 실행
3. insights.md에 결과 저장
```

**리서치 큐 등록:**
```markdown
# current/research_queue.md

## 대기 중인 주제
- [ ] AI 에이전트 트렌드 | 관련: 사업 | 제안일: 2024-01-06
- [ ] Stripe 수수료 비교 | 관련: 프레젠토 | 제안일: 2024-01-06
```

### 3. 선제적 알림

**JARVIS가 먼저 알려주는 것들**

| 알림 | 조건 | 예시 |
|------|------|------|
| 데드라인 임박 | D-3, D-7 | "프로젝트 X 3일 남았습니다" |
| 방치 프로젝트 | 7일+ 미활동 | "프로젝트 Y 10일간 방치" |
| 일정 충돌 | 겹치는 약속 | "오늘 2시, 3시 미팅 겹침" |
| 반복 패턴 | 3회+ 미룸 | "이 항목 3번째 미루시네요" |

### 4. 컨텍스트 연결

**프로필 기반 맞춤 조언**

JARVIS는 `profile.md`를 참조해서:
- 당신의 성향에 맞는 제안
- 과거 선호를 반영한 옵션
- 스트레스 패턴 고려한 분량 조절

**프로필 강화:**
```
"나는 아침형이야"
"긴 회의 싫어해"
"데드라인 있어야 움직여"
→ JARVIS가 자동 기록 & 활용
```

## 💡 팁 & 트릭

### 효과적인 사용법

#### 1. 구체적으로 요청하기

```
❌ "프로젝트 어떻게 돼?"
✅ "프레젠토 KCP 연동 상태 어때?"

❌ "리서치해줘"
✅ "/research AI 코딩 에이전트 시장 규모 2024"
```

#### 2. inbox 적극 활용

```
"[아이디어] 기록해둬"
→ 나중에 정리할 때까지 안전 보관
→ 잊어버려도 JARVIS가 리마인드
```

#### 3. 일일 기록 습관

```
# 하루 끝에
"오늘 뭐 했더라?"
"오늘 프레젠토 결제 연동 완료, 해동검도 피드백 받음"

→ weekly-log.md에 자동 기록
→ 주간 회고 시 유용
```

#### 4. 결정 전 리서치

```
중요한 결정 전:
/research [옵션A] vs [옵션B]

새로운 시도 전:
/chaos [목표] 달성 방법
```

### 자주 하는 실수

| 실수 | 해결책 |
|------|--------|
| 업데이트 안 함 | `projects.md`, `todo.md` 주기적 업데이트 |
| 너무 추상적 요청 | 구체적인 프로젝트명, 작업명 포함 |
| 기록 미루기 | 떠오를 때 바로 "기록해둬" |
| Skills 안 씀 | `/research`, `/chaos` 적극 활용 |

### 단축 표현

| 긴 표현 | 단축 |
|--------|------|
| "자세히 알아봐줘" | `/research --deep` |
| "빠르게 확인해줘" | `/research --quick` |
| "창의적으로 생각해봐" | `/chaos` |
| "다른 프로젝트에서" | `/pm [프로젝트]` |

## 📋 명령어 치트시트

### 세션 관리
```
와썹                    # 세션 시작
브리핑해줘              # 빠른 현황
오늘 뭐 해야 돼?        # 오늘 할 일
```

### 리서치 & 발상
```
/research [주제]        # 심층 리서치
/research [주제] --deep # 수렴까지
/research [주제] --quick # 빠른 스캔
/chaos [주제]           # 아이디어 발상
/chaos [주제] --wild    # 최대 창의성
```

### 프로젝트 관리
```
/pm [프로젝트] [지시]   # 작업 위임
/pm list               # 세션 목록
/pm status             # 전체 상태
```

### 마케팅
```
/marketing release [버전]    # 릴리즈 홍보
/marketing portfolio [프로젝트] # 포트폴리오
/marketing branding [주제]   # 개인 브랜딩
```

### 기록
```
[아이디어] 기록해둬     # inbox에 저장
[할일] 추가해줘        # todo에 추가
오늘 [내용] 했어       # 일일 기록
```

## 🆘 문제 해결

| 문제 | 해결 |
|------|------|
| JARVIS가 안 깨어남 | `와썹` 다시 입력 |
| 브리핑이 이상함 | `projects.md` 형식 확인 |
| Skills가 안 됨 | `.claude/skills/` 폴더 확인 |
| /pm이 안 됨 | tmux 세션 확인, `PM-GUIDE.md` 참조 |
| 기록이 안 됨 | `current/` 폴더 권한 확인 |
