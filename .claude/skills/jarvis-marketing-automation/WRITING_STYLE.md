# Tony님 필체 가이드 v1.0

> "군더더기 없이 코어에 가까운 거리. 밀도 있는 문장. 인간적 매력."

---

## 핵심 원칙

```
1. 반전 구조 - 예상을 비틈
2. 의인화 - 무생물에 캐릭터 부여
3. 짧은 문장 - 리듬감
4. 펀치라인 - 마지막에 위트 있는 결론
5. 자기 객관화 - 건조한 톤
```

---

## ❌ DON'T (AI 냄새)

### 금지 단어/표현

| 카테고리 | 금지 |
|---------|------|
| **AI 단어** | delve, tapestry, realm, landscape, leverage, robust, comprehensive |
| **전환어** | Furthermore, Moreover, In conclusion, Additionally |
| **시작 문구** | "오늘날의 빠르게 변화하는...", "Let's dive in" |
| **번역체** | ~되어진다, ~주어진다, ~것이다 |
| **이모지** | 3개 이상 금지, 🚀✨💪🎉 남발 금지 |

### 금지 구조

```
❌ 균일한 문단 길이
❌ 매 문장 느낌표
❌ 앞서 한 말 반복하며 정리
❌ "Overall,", "In summary," 로 끝내기
```

## ✅ DO (Tony님 스타일)

### 구조 패턴

**패턴 1: 반전 구조**
```
설정 (기대)
↓
현실 (반전)
↓
펀치라인 (통찰)
```

**패턴 2: 의인화**
```
무생물에 의도/감정 부여
↓
그 "행동"의 결과
↓
자기 객관화 결론
```

**패턴 3: Plan A/B**
```
Plan A: (이상적)
Plan B: (현실적)
↓
실제 결과: (둘 다 아님)
```

### 문장 기법

| 기법 | 예시 |
|------|------|
| **숫자로 시작** | "3일 만에 500명", "47개 알림" |
| **문장 길이 변화** | 짧게. 그리고 가끔은 좀 길게 풀어서. |
| **And/But 시작** | "근데 이게 함정.", "그리고 반전." |
| **질문형** | "그래서 뭘 했냐고?" |
| **대시로 끊기** | "결과—실패." |

### 이모지 규칙

```
- 최대 1-2개
- 끝에 포인트로만
- 감정 강조 아닌 아이러니용
- 😼🍯 (참고 예시처럼)
```

## 예시 비교

### ❌ AI 스타일 (금지)

```
오늘날의 빠르게 변화하는 기술 환경에서,
개발자들은 효율적인 배포 파이프라인을 구축하는 것이
매우 중요합니다. Furthermore, 자동화된 CI/CD 시스템을
활용함으로써 생산성을 크게 향상시킬 수 있습니다.
In conclusion, 이러한 접근 방식은 팀의 전반적인
퍼포먼스를 개선하는 데 도움이 됩니다! 🚀✨💪
```

### ✅ Tony님 스타일 (반전 구조)

```
배포 자동화 도입함.
매일 30분 절약된다고 좋아했음.

1주일 후 현실:
절약된 시간으로 새 기능 만듦.
그거 배포하느라 30분 씀.

시간은 절약되는 게 아니었다.
이동하는 거였다.
```

### ✅ Tony님 스타일 (의인화)

```
CI/CD 파이프라인 세팅 완료.
"이제 넌 자유야" 하고 퇴근함.

다음 날 아침.
슬랙 알림 47개.
파이프라인이 밤새 자유롭게 실패하고 있었다.

자동화는 일을 대신 해주는 게 아니다.
실패를 대신 해주는 거다.
```

## 채널별 톤 조절

### Twitter/X
```
가장 날카롭게.
짧게. 임팩트.
숫자 + 반전 + 한 줄 펀치라인.
```

### Threads
```
조금 더 풀어서.
대화하듯.
"~했는데" "~거든" 같은 구어체 OK.
```

### LinkedIn
```
같은 스토리, 비즈니스 래핑.
"이런 교훈을 얻었다" 대신
"이게 다음 프로젝트에 적용됐다"
```

### 네이버 블로그
```
구조는 유지하되 길이 확장.
중간중간 소제목.
마지막에 "결론" 대신 스토리로 마무리.
```

## 체크리스트 (게시 전)

```
□ 소리 내어 읽었을 때 자연스러운가?
□ "Furthermore", "Moreover" 없는가?
□ 이모지 2개 이하인가?
□ 반전 또는 펀치라인 있는가?
□ 첫 문장이 훅이 되는가?
□ 마지막 문장이 기억에 남는가?
```

## 참고: 좋은 예시 구조

Tony님이 좋아한 "꿀 훔치기" 글 구조:

```
1. 설정 (mission)
   → "3 brave friends set out on a serious mission"

2. Plan A/B (대비)
   → "Smile at the bees" / "Run very fast"

3. 반전 (예상 못한 표현)
   → "bees don't negotiate"

4. 제3자 시점 (캐릭터)
   → 고양이: "I warned you. I always warn you."

5. 펀치라인 (비유적 마무리)
   → "swollen pride, itchy memories"
```
