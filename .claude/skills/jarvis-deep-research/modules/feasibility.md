# Feasibility Module - 실행 가능성 필터

## Overview

일반적인 리서치 결과를 **사용자 맥락**에 맞게 필터링하여 실행 가능한 인사이트로 변환.

---

## 사용자 맥락 자동 로드

### profile.md에서 로드되는 항목

```yaml
# 자동 주입되는 사용자 맥락
user_context:
  role: "1인 개발자"
  goal: "월 2,000만원 패시브 인컴"
  tech_stack:
    - Python
    - Playwright
    - AI 활용
  constraints:
    capital: "제한적"
    time: "제한적"
    team: "1인"
  preferences:
    - 자동화 우선
    - 작게 시작
    - 빠른 검증
```

### insights.md에서 로드되는 제외 목록

```yaml
# 이미 검토 후 패스한 아이템
exclusion_list:
  - "CPA/콘텐츠 락킹/틱톡 파이프라인"
    reason: "완전 자동화 불가"
    date: "2025-12-03"
  # 추가 항목...
```

## 실행 가능성 평가 매트릭스

### 필수 평가 항목 (8개)

| 항목 | 설명 | 평가 기준 |
|------|------|----------|
| **초기 비용** | 시작에 필요한 자금 | $0-100 / $100-1K / $1K+ |
| **1인 실행** | 혼자 운영 가능 여부 | Y / N / Partial |
| **자동화 범위** | 자동화 가능 비율 | % |
| **기술 적합성** | 보유 기술로 구현 가능 | Y / N / 학습필요 |
| **첫 수익까지** | 수익 발생 예상 기간 | 즉시 / 1주 / 1개월 / 3개월+ |
| **예상 ROI** | 투자 대비 수익률 | % (연간) |
| **리스크 레벨** | 실패 시 손실 규모 | L / M / H |
| **시간 투자** | 주당 필요 시간 | 시간/주 |

### 평가 템플릿

```markdown
## 실행 가능성 평가

| 항목 | 평가 | 점수 | 비고 |
|------|------|------|------|
| 초기 비용 | $[금액] | /10 | [세부] |
| 1인 실행 | Y/N/P | /10 | [세부] |
| 자동화 범위 | [X]% | /10 | [세부] |
| 기술 적합성 | Y/N/L | /10 | [세부] |
| 첫 수익까지 | [기간] | /10 | [세부] |
| 예상 ROI | [X]% | /10 | [세부] |
| 리스크 레벨 | L/M/H | /10 | [세부] |
| 시간 투자 | [X]h/주 | /10 | [세부] |

### 종합 점수: [XX]/80
```

## 점수 산정 기준

### 초기 비용 (Capital)

| 금액 | 점수 | 설명 |
|------|------|------|
| $0 | 10 | 무료 시작 |
| $1-50 | 8 | 거의 무료 |
| $50-200 | 6 | 저비용 |
| $200-1000 | 4 | 중간 비용 |
| $1000+ | 2 | 고비용 |

### 1인 실행 (Solo Operation)

| 평가 | 점수 | 설명 |
|------|------|------|
| Y (Yes) | 10 | 완전히 혼자 가능 |
| P (Partial) | 6 | 일부 외주/협업 필요 |
| N (No) | 2 | 팀 필수 |

### 자동화 범위 (Automation)

| 비율 | 점수 | 설명 |
|------|------|------|
| 90-100% | 10 | 완전 자동화 |
| 70-89% | 8 | 대부분 자동화 |
| 50-69% | 6 | 반자동화 |
| 30-49% | 4 | 수동 개입 많음 |
| 0-29% | 2 | 대부분 수동 |

## 판정 기준

### 종합 점수 기반 판정

| 점수 | 판정 | 설명 |
|------|------|------|
| 65-80 | **Go** | 강력 추천, 즉시 실행 |
| 50-64 | **Conditional** | 조건부 추천, 특정 조건 충족 시 |
| 35-49 | **No-Go (Review)** | 비추천, 재검토 필요 |
| 0-34 | **No-Go (Hard)** | 강력 비추천 |

### 절대적 No-Go 조건 (Override)

아래 조건 중 하나라도 해당 시 점수와 무관하게 No-Go:

```markdown
## 절대 No-Go 조건
- [ ] 자동화 범위 < 50% (형님 선호 위반)
- [ ] 1인 실행 = N (팀 필수)
- [ ] 기술 적합성 = N (완전 새 스택)
- [ ] 리스크 = H AND 초기 비용 > $1000
- [ ] 이미 제외 목록에 있음
```

### Conditional 판정 시 조건 명시

```markdown
## Conditional 조건

### 충족 시 Go로 변경
1. [조건 1] - 현재: [상태], 필요: [목표]
2. [조건 2] - 현재: [상태], 필요: [목표]

### 검증 방법
- [조건 1 검증 방법]
- [조건 2 검증 방법]
```

- [요약 1]

- [요약 3]

## 사용자 맥락 커스터마이징

### profile.md 업데이트 시 반영

사용자 상황 변화 시 필터 기준 자동 조정:

```yaml
# 예: 자금 확보 시
constraints:
  capital: "여유 있음"  # "제한적" → "여유 있음"

# 결과: 초기 비용 점수 기준 완화
```

### 임시 오버라이드

특정 리서치에서만 기준 조정:

```
/research {주제} --capital-limit $5000 --time-ok 20h
```
